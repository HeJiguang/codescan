# Security Policy

## Reporting a Vulnerability

If you believe you found a real vulnerability in CodeScan itself, please do not open a public GitHub issue with exploit details first.

Instead, send a private report to:

- `sinwtao@outlook.com`

Please include:

- a clear description of the issue
- affected files or components
- reproduction steps or a proof of concept
- impact assessment if known

## Scope

This policy is for vulnerabilities in CodeScan itself, including:

- the scanner runtime
- MCP server behavior
- bundled skill or installation guidance
- report generation or unsafe output handling

This policy is not for vulnerabilities discovered by CodeScan in third-party repositories. Those should be reported to the maintainers of the affected project.

## Response Expectations

The goal is to acknowledge legitimate reports quickly, reproduce them, and patch them before public disclosure when reasonable.

## Safe Disclosure

Please avoid publishing working exploit details before a fix or mitigation is available.
