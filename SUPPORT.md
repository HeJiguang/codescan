# Support

Use the channel that matches the kind of help you need.

## Questions And Setup Help

Use GitHub Discussions for:

- installation questions
- MCP or Codex setup help
- provider or model configuration questions
- "is this expected?" behavior checks
- workflow discussions and usage examples

## Bugs

Use the GitHub bug report template when you can reproduce a problem.

Include:

- operating system
- Python version
- whether you used CLI, MCP, GUI, or Skill flow
- provider and model name if AI analysis was involved
- exact command or workflow
- logs, screenshots, or example files when relevant

## Feature Requests

Use the feature request template for focused product or workflow proposals.

Good requests usually explain:

- who benefits
- what workflow improves
- whether it is CLI, MCP, GUI, Skill, rule quality, or reporting related
- what a practical first version would look like

## Security Reports

Do not open public issues for undisclosed vulnerabilities.

Use the process described in [SECURITY.md](SECURITY.md).

## Contribution Paths

If you want to help directly, start here:

- [Contributing Guide](docs/CONTRIBUTING.md)
- [Good First Issues](docs/good-first-issues.md)
- [Community Guide](docs/community.md)

## What Maintainers Need Most

The highest-value reports for CodeScan usually include one of these:

- a concrete false positive with file context
- a missed security case that should have been detected
- MCP or Codex workflow friction with exact steps
- packaging or onboarding pain that blocked a real first run
- a real repository or diff where the current result shape is hard to use
